# Historical Trend Charts — v1.3.0

> Visualize rating, rank, and review changes for tracked apps over time.

## Current State

- **Backend:** Partial — chart snapshots and review data are stored, but no trend endpoints
- **Frontend:** Missing

## Tasks

### 1. Backend — Trend Endpoints

#### 1.1 Rating Trend

```
GET /apps/{platform}/{externalId}/trends/rating
?period=30d|90d|1y
```

Response: `[{ date, rating, rating_count }]`

#### 1.2 Chart Rank Trend

```
GET /apps/{platform}/{externalId}/trends/rank
?collection=top_free&country=tr&period=30d
```

Response: `[{ date, rank }]`

#### 1.3 Review Volume Trend

```
GET /apps/{platform}/{externalId}/trends/reviews
?period=30d
```

Response: `[{ date, count, average_rating }]`

### 2. Frontend — Trend Charts

Add to **App Show page** (`web/src/pages/apps/Show.tsx`) as tab or section:

- Rating trend line chart (recharts or chart.js)
- Chart rank trend (inverted Y axis — 1 at top)
- Review volume bar chart
- Period selector (30d, 90d, 1y)
