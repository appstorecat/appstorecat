# Notification System — v1.2.0

> Notification infrastructure for changes detected in tracked apps and competitors.

## Current State

- **Backend:** Missing — no notification model, channels, or preferences
- **Frontend:** Missing
- **MCP:** Missing

## Tasks

### 1. Backend — Notification Infrastructure

#### 1.1 Migration & Model

- `notifications` table (can use Laravel built-in `DatabaseNotification`)
- `notification_preferences` table (per-user channel/type preferences)

#### 1.2 Notification Classes

- `AppChangedNotification` — tracked app listing change
- `CompetitorChangedNotification` — competitor app change
- `NewReviewNotification` — new review received (optional, high volume)

#### 1.3 Channels

- **Database** — notification bell in web UI
- **Email** — summary email (daily/weekly digest)
- **Webhook** — external integration (Slack, Discord, etc.)

#### 1.4 Preference Endpoints

```
GET    /account/notification-preferences
PUT    /account/notification-preferences
```

#### 1.5 Notification Endpoints

```
GET    /notifications              → list (paginated)
POST   /notifications/read-all     → mark all as read
DELETE /notifications/{id}         → delete single notification
```
