# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 0.0.3   | Yes                |
| < 0.0.3 | No                 |

## Reporting a Vulnerability

If you discover a security vulnerability in AppStoreCat, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email: **ismailcakirnet@gmail.com**

Include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fix (optional)

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial Assessment:** Within 1 week
- **Fix & Disclosure:** Coordinated with reporter

## Scope

This policy applies to all services in the AppStoreCat monorepo:

- `backend/` — Laravel API
- `frontend/` — React SPA
- `scraper-appstore/` — App Store scraper microservice
- `scraper-gplay/` — Google Play scraper microservice

## Out of Scope

- Vulnerabilities in third-party dependencies (report upstream)
- Issues in development/test environments only
- Social engineering attacks

Thank you for helping keep AppStoreCat safe.
